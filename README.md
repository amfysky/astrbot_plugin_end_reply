# 屏蔽回复

拦截并丢弃模型的一切纯文本输出，使其**只能通过工具(tool)向用户发消息**。

## 解决什么问题

在本地 Agent 模式下，模型「不带工具调用的那一段正文」会被 AstrBot 当作普通回复自动下发。如果你希望机器人**只通过工具发消息**（例如自定义的「消息发送」工具），就需要把模型直接写出的纯文本拦下来，否则它会绕过工具直接回复用户。

本插件把模型的纯文本回复在下发前全部丢弃，于是「直接写正文」这条路被彻底掐断，内容只能经由工具发送。

## 工作原理

- **拦截文本输出（`on_decorating_result`）**：当结果是「大模型直接回复」(`is_llm_result`) 时，剔除其消息链里的全部文本(`Plain`)组件。链被清空后发送阶段判定为空消息、不再下发。命令回复、工具用 `event.send` 直发的内容都不是 `LLM_RESULT`，不受影响。这里只拦「发送」，会话历史里仍保留模型原始正文。
- **预注入规则（`on_llm_request`）**：把「纯文本会被丢弃、只能用工具发消息」这条约定追加进 `system_prompt`，明确告诉模型该用工具发消息。带标记前缀做幂等，避免多轮里重复追加。

## 局限

流式输出(`streaming_response`)下，正文是边生成边逐块下发的，在结果装饰阶段之前就已发出，本拦截对流式模式无效。**如需在流式下生效，请关闭流式输出。**

## 安装

把本仓库文件夹放进 AstrBot 的 `data/plugins/` 目录，在管理面板重载插件即可。无第三方依赖。
